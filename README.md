CoreApp - Documentación de Arquitectura de Software (DAS)
Este repositorio centraliza la documentación técnica y evolutiva de CoreApp, una solución integral diseñada para modernizar la gestión administrativa del edificio "El Mirador".

📋 Descripción del Proyecto
CoreApp surge como respuesta a la necesidad de centralizar las operaciones financieras, la comunicación entre residentes y la gestión de áreas comunes en una sola plataforma. Su objetivo es reducir la burocracia, eliminar errores de cálculo financiero y restaurar la confianza en la administración del edificio mediante un sistema auditable, escalable y seguro.

🏛️ Sobre el DAS (Documento de Arquitectura de Software)
El DAS es el núcleo de este repositorio. Define los cimientos técnicos sobre los cuales se construye el prototipo. Sus pilares incluyen:

Visión del Sistema: Definición precisa de Requerimientos Funcionales (RF) y No Funcionales (RNF).

Modelo 4+1 Vistas: Estructura que permite visualizar el sistema desde diferentes perspectivas (lógica, procesos, implementación, física y casos de uso).

Calidad ISO/IEC 25010: Nuestras decisiones arquitectónicas están alineadas con los atributos de calidad medibles: Precisión (cálculos financieros al 0.00% de error), Usabilidad (interfaz intuitiva para todas las edades) y Disponibilidad.

Seguridad: Implementación de control de acceso basado en roles (Administrador, Empleado, Residente) para proteger datos sensibles.

🛠️ Prácticas de Desarrollo y Control
Prototipado: El diseño se ha validado mediante herramientas como Figma, iterando constantemente bajo heurísticas de usabilidad.

Control de Versiones (Secuencial): * Hemos adoptado un control de versiones secuencial (v1.1, v3.0.3, etc.).

Justificación: Esta práctica permite una trazabilidad lineal y cronológica de los hitos del semestre. Facilita la auditoría del progreso y evita la complejidad innecesaria de ramas (branches) paralelas, asegurando que cada entrega sea un reflejo claro de las correcciones y mejoras aplicadas tras el feedback docente.
